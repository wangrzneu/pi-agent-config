# External Projects

Use this guide when work depends on an open-source project that is not available locally.

1. Record the project URL and the version, tag, or commit in scope.
2. Use DeepWiki only as an orientation aid when available.
3. Verify important claims against source code, official documentation, tests, and release notes.
4. Prefer primary sources and link evidence to the exact version in scope.
5. Separate verified facts from inference and note unresolved uncertainty.

Do not upload private or sensitive repositories to third-party indexing services. For private projects, use approved local tools and repository access only.
